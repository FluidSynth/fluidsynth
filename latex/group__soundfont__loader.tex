\doxysubsubsection{Detailed Description}
Create custom Sound\+Font loaders

It is possible to add new Sound\+Font loaders to the synthesizer. This API allows for virtual Sound\+Font files to be loaded and synthesized, which may not actually be Sound\+Font files, as long as they can be represented by the Sound\+Font synthesis model.

To add a new Sound\+Font loader to the synthesizer, call fluid\+\_\+synth\+\_\+add\+\_\+sfloader() and pass a pointer to an \doxylink{group__Types_gad3e880056d2262e48468719757b3f5e6}{fluid\+\_\+sfloader\+\_\+t} instance created by \doxylink{group__soundfont__loader_ga9a1412c90ac4a1f945985d1bc1e51607}{new\+\_\+fluid\+\_\+sfloader()}. On creation, you must specify a callback function {\ttfamily load} that will be called for every file attempting to load it and if successful returns a \doxylink{group__Types_gaa6c18288f76608acbb10b80a153f4ab8}{fluid\+\_\+sfont\+\_\+t} instance, or NULL if it fails.

The \doxylink{group__Types_gaa6c18288f76608acbb10b80a153f4ab8}{fluid\+\_\+sfont\+\_\+t} structure contains a callback to obtain the name of the Sound\+Font. It contains two functions to iterate though the contained presets, and one function to obtain a preset corresponding to a bank and preset number. This function should return a \doxylink{group__Types_ga985e5ee05f433da841127750f67a4723}{fluid\+\_\+preset\+\_\+t} instance.

The \doxylink{group__Types_ga985e5ee05f433da841127750f67a4723}{fluid\+\_\+preset\+\_\+t} instance contains some functions to obtain information from the preset (name, bank, number). The most important callback is the noteon function. The noteon function is called by fluidsynth internally and should call fluid\+\_\+synth\+\_\+alloc\+\_\+voice() for every sample that has to be played. fluid\+\_\+synth\+\_\+alloc\+\_\+voice() expects a pointer to a \doxylink{group__Types_gabf9174d452679ca1a4ee7d693fb773cf}{fluid\+\_\+sample\+\_\+t} instance and returns a pointer to the opaque \doxylink{group__Types_ga5123ae5ef2d7806475267380c33604c3}{fluid\+\_\+voice\+\_\+t} structure. To set or increment the values of a generator, use fluid\+\_\+voice\+\_\+gen\+\_\+set() or fluid\+\_\+voice\+\_\+gen\+\_\+incr(). When you are finished initializing the voice call fluid\+\_\+voice\+\_\+start() to start playing the synthesis voice.
