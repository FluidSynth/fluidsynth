\doxysubsection{Detailed Description}
Logging interface

The default logging function of the fluidsynth prints its messages to the stderr. The synthesizer uses five level of messages\+: \doxylink{group__logging_ggaa341232c15addd9c336131984840ed9ea62d9ddd89d0a853606ce38d195daa99a}{FLUID\+\_\+\+PANIC}, \doxylink{group__logging_ggaa341232c15addd9c336131984840ed9eabc0ff48d192b3b0fc721946ccf095d25}{FLUID\+\_\+\+ERR}, \doxylink{group__logging_ggaa341232c15addd9c336131984840ed9ea394460be1ec74fce15e1ff683606df0f}{FLUID\+\_\+\+WARN}, \doxylink{group__logging_ggaa341232c15addd9c336131984840ed9eadfa902517702f2dd8ae849b810175ac4}{FLUID\+\_\+\+INFO}, and \doxylink{group__logging_ggaa341232c15addd9c336131984840ed9ea5542b67c606598020b832c31729175fc}{FLUID\+\_\+\+DBG}.

A client application can install a new log function to handle the messages differently. In the following example, the application sets a callback function to display \doxylink{group__logging_ggaa341232c15addd9c336131984840ed9ea62d9ddd89d0a853606ce38d195daa99a}{FLUID\+\_\+\+PANIC} messages in a dialog, and ignores all other messages by setting the log function to NULL\+:

\begin{DoxyNote}{Note}
The logging configuration is global and not tied to a specific synthesizer instance. That means that all synthesizer instances created in the same process share the same logging configuration. 
\end{DoxyNote}
